# دليل النشر والتسليم الكامل — نموذج متعدد المحلات

المرجع العربي الرسمي لنشر النظام وبيعه لعدة محلات خياطة.
(النسخة الفرنسية المختصرة: `DEPLOYMENT.md` — عند التعارض هذا الدليل هو الأحدث.)

**المعمارية:** نسخة معزولة لكل محل (قاعدة بيانات + حاويات مستقلة)، كلها على
VPS واحد خلف nginx. الأندرويد عبر APK موقّع، والآيفون واللابتوب عبر نسخة
الويب في المتصفح.

```
                 ┌────────────────── VPS واحد ──────────────────┐
 هاتف أندرويد ──▶│ nginx (HTTPS)                                 │
 آيفون/لابتوب ──▶│  ├─ api.rayan.<دومينك>  → حاوية محل 1 (:3001) │
                 │  ├─ app.rayan.<دومينك>  → ويب محل 1 (ملفات)   │
                 │  ├─ api.salon2.<دومينك> → حاوية محل 2 (:3002) │
                 │  └─ …                                          │
                 └───────────────────────────────────────────────┘
```

## المرحلة صفر: تجهيزات لمرة واحدة (تخدم كل المحلات القادمة)

### 0.1 شراء الدومين

- اشترِ **دومين رئيسي واحد فقط** باسم تجاري عام لخدمتك (وليس باسم محل
  بعينه — لأنك ستبيع لمحلات بأسماء مختلفة). من مسجّل موثوق: Namecheap،
  Porkbun، Squarespace…
- **يُفضَّل `.com` أو `.app`** على النطاقات المجانية السمعة سابقاً (مثل `.ml`
  المجاني قديماً) — بعض فلاتر التصفح/البريد تتشدد معها، وفرق السعر بسيط.
- لكل محل جديد أنشئ **subdomain مجانياً** تحت نفس الدومين — سجلّا **A** في
  إعدادات DNS يشيران إلى IP الخادم:

  ```
  api.rayan.<دومينك>    A    <IP الخادم>
  app.rayan.<دومينك>    A    <IP الخادم>
  ```

### 0.2 استئجار الـ VPS

- Droplet على DigitalOcean (نفس أسلوب EduGete): **2GB RAM** تكفي من محلّين
  إلى ~خمسة محلات، وترقّى لاحقاً بضغطة زر.
- ثبّت Docker + Docker Compose.
- **جدار الحماية:** افتح فقط 22 (SSH) و80 و443. **لا تفتح أبداً** منافذ الـ API
  (3001، 3002…) ولا PostgreSQL (5432) للعامة — تبقى داخلية خلف nginx:

  ```bash
  ufw allow 22 && ufw allow 80 && ufw allow 443 && ufw enable
  ```

- ثبّت **nginx** (موجّه رئيسي لكل الـ subdomains) و**certbot** (شهادات HTTPS
  مجانية تتجدد تلقائياً):

  ```bash
  apt install nginx certbot python3-certbot-nginx
  ```

### 0.3 هيكلة مجلدات السيرفر

```
/srv/
  rayan-couture/   ← محل 1 (نسخة كاملة من المشروع + .env خاص)
  salon-2/         ← محل 2 (لاحقاً)
  salon-3/         ← …
```

كل مجلد = نسخة مستقلة من **نفس الكود**، ببيانات ومنافذ وأسرار مختلفة.
(اسم المجلد يصبح تلقائياً بادئة أسماء الحاويات والـ volumes في Docker —
فلا تصادم بين المحلات.)

### 0.4 مفتاح توقيع الأندرويد (مرة واحدة، على جهازك)

```powershell
keytool -genkey -v -keystore C:\keys\upload-keystore.jks -keyalg RSA -keysize 2048 -validity 10000 -alias upload
```

ثم انسخ `tailoring_app/android/key.properties.example` إلى
`android/key.properties` واملأه (المسار وكلمات السر). الملف مستثنى من git.

> 🔑 **ملف `.jks` وكلمات سرّه = عمرها عمر المشروع.** فقدانها يعني استحالة
> تحديث أي تطبيق مُسلَّم. مفتاح واحد يكفي لتوقيع كل المحلات.

## المرحلة 1: نشر المحل الأول — خطوة بخطوة

1. **استنسخ المشروع:**

   ```bash
   cd /srv && git clone <رابط مستودعك> rayan-couture && cd rayan-couture
   ```

2. **أنشئ `.env` خاصاً بالمحل** (انطلاقاً من `backend/.env.example`):

   ```
   DB_PASSWORD=<عشوائي قوي>
   JWT_SECRET=<عشوائي طويل — فريد لهذا المحل حصراً>
   API_PORT=3001                 ← منفذ فريد لكل محل (3002 للثاني…)
   SHOP_NAME=Rayan Couture
   DEFAULT_PIECE_RATE=0
   PROMO_GROUP_LINK=
   SEED_MANAGER_USERNAME=gerant
   SEED_MANAGER_PASSWORD=<قوي مؤقت — يُغيَّر عند أول دخول>
   SEED_SECRETARY_USERNAME=secretaire
   SEED_SECRETARY_PASSWORD=<قوي مؤقت>
   ```

3. **شغّل وجهّز المحل:**

   ```bash
   docker compose up -d --build
   docker compose exec api npm run setup-shop
   ```

   (`setup-shop` = الهجرات + الحسابان + هوية المحل، وآمن لإعادة التشغيل.)

4. **أعدّ nginx لهذا المحل** — `/etc/nginx/sites-available/rayan`:

   ```nginx
   server {
       server_name api.rayan.<دومينك>;
       location / {
           proxy_pass http://127.0.0.1:3001;      # = API_PORT
           proxy_set_header Host $host;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
           proxy_set_header X-Forwarded-Proto $scheme;
           client_max_body_size 20M;              # رفع الصور/الفيديو
       }
   }
   server {
       server_name app.rayan.<دومينك>;
       root /var/www/rayan-web;
       location / { try_files $uri $uri/ /index.html; }
   }
   ```

   ```bash
   ln -s /etc/nginx/sites-available/rayan /etc/nginx/sites-enabled/
   nginx -t && systemctl reload nginx
   ```

5. **أصدر شهادة HTTPS:**

   ```bash
   certbot --nginx -d api.rayan.<دومينك> -d app.rayan.<دومينك>
   ```

   اختبار: `https://api.rayan.<دومينك>/api/settings/public` يرجع JSON المحل.

6. **ابنِ نسخة الويب** (على جهازك) وارفعها:

   ```bash
   cd tailoring_app
   flutter build web --release --dart-define=API_URL=https://api.rayan.<دومينك>
   # ثم انقل build/web إلى /var/www/rayan-web على الخادم (scp أو rsync)
   ```

7. **ابنِ APK موقّعاً باسم المحل.** عدّل أولاً السطرين في
   `tailoring_app/android/gradle.properties`:

   ```properties
   appName=Rayan Couture
   appId=com.rayancouture.app
   ```

   ثم:

   ```bash
   flutter build apk --release --dart-define=API_URL=https://api.rayan.<دومينك>
   ```

   الناتج: `build/app/outputs/flutter-apk/app-release.apk`.

   > ⚠️ يجب أن يمر عبر `gradle.properties` — متغيرات البيئة
   > `ORG_GRADLE_PROJECT_*` لا تصل إلى Gradle عبر `flutter build`
   > (مُجرَّب: الـ APK يخرج بالقيم الافتراضية).

8. **فعّل النسخ الاحتياطي التلقائي** — cron يومي (مثلاً 3 فجراً) يحفظ
   **ثلاثة أشياء** ويرسلها **خارج الـ VPS** (تخزين سحابي منفصل — نفس القرص
   لا يكفي):
   - قاعدة البيانات: `docker compose exec -T db pg_dump -U couture couture_mali > backup_$(date +%F).sql`
   - مجلد الرفوعات (صور المنتجات والشعارات — لا تُسترجع من القاعدة):
     volume الـ `uploads_data`
   - (على جهازك) ملف `.jks` وكلمات سرّه — نسخة في مكانين على الأقل

9. **اختبر بنفسك بالكامل قبل التسليم:** دخول بالحسابين، جرّب كل وحدة،
   تأكد أن السكريتار **لا ترى Finances** (جرّب بحسابها فعلاً)، تأكد أن
   الفاتورة PDF وواتساب يعملان من هاتف حقيقي عبر HTTPS.

## المرحلة 2: التسليم الفعلي لصاحب المحل

سلّمه **ثلاثة أشياء** في جلسة واحدة، وجهاً لوجه إن أمكن:

1. **ملف APK** (واتساب أو نقل مباشر) — يثبّته على هواتف المحل. أول تثبيت
   يحتاج السماح بـ«تثبيت تطبيقات غير معروفة» للتطبيق الذي فُتح منه الملف.
2. **رابط الويب** `https://app.rayan.<دومينك>` للابتوب والآيفون — علّمه
   «إضافة إلى الشاشة الرئيسية» في Safari و«Install app» في Chrome لتجربة
   أقرب لتطبيق حقيقي.
3. **ورقة الحسابات:** بيانات حساب المدير وحساب السكريتير المؤقتة، مع
   تعليمة صريحة: **«غيّر كلمتي المرور فوراً من الإعدادات عند أول دخول»**
   (الميزة داخل التطبيق: Paramètres → تغيير كلمة المرور/اسم المستخدم).

ثم:

- **جولة تدريب 15–20 دقيقة** على الأساسيات: إضافة عميل + مقاساته، إنشاء
  طلب، تسجيل بيع، رفع الشعار من الإعدادات، الفاتورة وإرسالها واتساب.
- **اطلب تأكيداً كتابياً بسيطاً** (حتى رسالة واتساب) بأنه استلم واختبر
  التطبيق — يحميك عند أي نزاع لاحق.
- إن نسي المدير كلمة مروره لاحقاً، أنت فقط تعيد تعيينها من الخادم:

  ```bash
  docker compose exec api npm run reset-password
  ```

## المرحلة 3: تكرار العملية للمحل الثاني وما بعده

نفس خطوات المرحلة 1 بالضبط، مع تغيير هذه فقط:

| العنصر | محل 2 (مثال) |
| --- | --- |
| المجلد | `/srv/salon-2` |
| `API_PORT` في `.env` | `3002` |
| `JWT_SECRET` و`DB_PASSWORD` وكلمات السر | **جديدة كلياً — لا يُعاد استخدام أي سر** |
| `SHOP_NAME` | Atelier Diallo |
| الـ subdomains + بلوك nginx + certbot | `api.salon2.<دومينك>` و`app.salon2.<دومينك>` |
| نسخة الويب | تُبنى بـ `API_URL` الجديد → `/var/www/salon2-web` |
| الـ APK | عدّل `appName=Atelier Diallo` و`appId=com.atelierdiallo.app` ثم ابنِ بنفس المفتاح |

> ⚠️ **تعديل `gradle.properties` قبل بناء APK كل محل ليس اختيارياً**: بدونه
> يخرج التطبيق باسم ومعرّف المحل الأول، ولن يمكن تثبيت تطبيقَي محلّين على
> هاتف واحد. وبعد تسليم محل **لا تغيّر `appId` الخاص به أبداً**.

**قاعدتان ذهبيتان:**

- **لا تعدّل الكود لأجل محل معيّن أبداً.** أي تخصيص (اسم، شعار، تعرفة،
  ألوان، رابط) يكون عبر إعدادات التطبيق أو `.env` أو `gradle.properties` —
  وليس فرع git منفصلاً. هذه أهم قاعدة تحميك من كابوس الصيانة.
- **جدول مركزي** (Excel/Google Sheet) لكل محل: الاسم، الـ subdomains،
  المنفذ، `appId`، تاريخ التسليم، نسخة الـ APK المسلَّمة، آخر تحديث،
  حالة الدفع.

## المرحلة 4: الصيانة وطلبات الميزات المستقبلية

**القاعدة: لا تطوير على الإنتاج أبداً.** الدورة:

1. طوّر واختبر **محلياً** (`npm run dev` + `flutter run`).
2. **`npm test` يجب أن يبقى أخضر** — مجموعة الاختبارات تثبت آلياً عزل
   السكريتار المالي وقواعد append-only؛ نجاحها شرط لأي نشر.
3. `git commit` + `git push`.
4. **تحديث محل حي:**

   ```bash
   cd /srv/<المحل>
   docker compose exec -T db pg_dump -U couture couture_mali > pre_update_$(date +%F).sql   # احتياط أولاً
   git pull
   docker compose up -d --build        # الهجرات تعمل تلقائياً عند الإقلاع
   ```

   - نسخة الويب: أعد البناء وانسخ `build/web` — المستخدم يحدّث الصفحة فقط.
   - الـ APK: فقط إن تغيّرت الواجهة/المنطق — ارفع الرقم في `pubspec.yaml`
     (`version: 1.0.1+2`)، ابنِ **بنفس المفتاح ونفس `appId`**، أرسله —
     يُثبَّت فوق القديم والبيانات محفوظة.
   - **الترتيب دائماً: الخادم أولاً، ثم التطبيق.**

5. **ميزة عامة تفيد الجميع:** طوّرها مرة، حدّث كل المحلات بنفس الأمرين.
6. **ميزة خاصة بمحل يدفع مقابلها:** صمّمها **إعداداً قابلاً للتفعيل**
   (صف في جدول `settings` لقاعدة ذلك المحل فقط — مثل `theme_color`
   و`custom_garments` الموجودتين) — لا نسخة كود منفصلة أبداً.
7. **تغيير مخطط قاعدة البيانات:** ملف **جديد** في `backend/migrations/`
   فقط — لا يُعدَّل ملف مُطبَّق. وأي جدول مالي جديد يلتزم بمبدأ
   append-only + جدول تصحيحات + اختبارات في نفس الـ commit.

## قائمة التحقق النهائية قبل تسليم أي محل جديد

- [ ] `JWT_SECRET` و`DB_PASSWORD` وكلمات السر الأولية **فريدة لهذا المحل**
- [ ] HTTPS يعمل (لا تحذير في المتصفح، والأندرويد يتصل وينجح الدخول)
- [ ] المنافذ الداخلية (API/DB) غير مفتوحة للعامة (`ufw status` للتأكد)
- [ ] النسخ الاحتياطي التلقائي مُفعَّل **ومُختبَر باسترجاع تجريبي** مرة
      واحدة على الأقل، ويشمل القاعدة **و`uploads`**، ويُرسَل خارج الـ VPS
- [ ] حسابا المدير والسكريتير يعملان، وعزل Finances مُختبَر **يدوياً
      بحساب السكريتير**
- [ ] APK موقَّع بالمفتاح الرسمي (ليس debug)، وبـ`appName`/`appId`
      الصحيحين لهذا المحل
- [ ] الشعار واسم المحل صحيحان في: شاشة الدخول، لوحة التحكم، الفاتورة PDF
- [ ] `promo_group_link` والتعرفة الافتراضية مضبوطتان (أو متروكتان عمداً)
- [ ] الجدول المركزي محدَّث بمعلومات هذا المحل
- [ ] تأكيد استلام كتابي من صاحب المحل + تغيير كلمات المرور أمامك
